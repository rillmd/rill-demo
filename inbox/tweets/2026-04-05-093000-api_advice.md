If your API still returns `{ "error": "Something went wrong" }` in 2026, we need to talk.

Typed errors with stable codes are table stakes. Your clients can't write retry logic against a sentence. They can write it against `RATE_LIMITED`, `VALIDATION_FAILED`, `UPSTREAM_TIMEOUT`. Pick a shape. Document it. Version it.
