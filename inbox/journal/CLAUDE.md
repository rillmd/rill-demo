# inbox/journal/ — Shallow Think

Instant thought capture for the user. Microfile strategy.

## Filename

`YYYY-MM-DD-HHmmss.md` (e.g., `2026-02-13-105830.md`)

Auto-generated by `rill log "text"`. Do not create manually.

## Frontmatter

```yaml
---
created: 2026-02-13T10:58+09:00
---
```

Only `created` is required. Automatically set by `rill log`.

## Microfile Strategy

- **1 file = 1 input**
- Record thoughts immediately as they come
- Short is fine (even a single sentence)
- Separate distinct thoughts into separate files

## Immutability Principle

- **Never modify existing files** (no appending either)
- Corrections or additional information go into **new files**
- Use Markdown links in the body text if cross-referencing is needed

## `_organized/`

`/distill` generates organized versions:
```yaml
---
created: 2026-02-13T10:58+09:00
organized: true
---
```

## Why This Rule

The immutability of inbox/journal/ underpins Rill's entire transaction log design. Allowing appends would break `.processed` tracking, `/distill` idempotency, and the correspondence with `_organized/` files.
