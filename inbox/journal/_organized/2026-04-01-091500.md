## LLM Context Window Management Patterns

Observation that naive "stuff everything in" approaches to LLM context windows lead to degraded instruction following. Proposed a tiered context management strategy.

### Proposed Architecture

- **Pinned zone**: System prompt and critical instructions always retained at full fidelity
- **Priority queue (middle)**: Managed dynamically based on recency and relevance
  - Recent messages (last 3 turns): full fidelity
  - Older messages (3-10 turns): summarized
  - Beyond 10 turns: evicted
- **Criticality heuristic**: User preferences > conversation state > previous tool outputs > general history

### Hybrid with RAG

RAG externalizes the context management problem for reference material. A hybrid approach could use RAG for reference lookups while applying careful sliding-window management for conversation state. This separates "what the model needs to remember" from "what the model needs to access."

### Open Questions

- How to determine criticality thresholds dynamically?
- Should summarization be done by the same model or a smaller one?
- What's the quality degradation curve as context gets more aggressively compressed?
