Sunday afternoon. Thinking about a side project I want to start: a small LLM eval harness for the team.

Not for the API v2 work directly. The trigger is that we have maybe four or five AI-assisted features in the product now — the copilot autocomplete, the error explainer, the doc search, the onboarding walkthrough — and every time someone tweaks a prompt, we have no way to tell if it got better or worse. We're flying on vibes.

What I'd want: a dead-simple harness where you drop in a prompt, a set of test inputs, and a rubric, and it spits out a score matrix you can diff across prompt versions. Nothing fancy. No MLflow, no hosted platform, no tracking server. Just a CLI that reads YAML and writes a Markdown report.

It feels like a "yak shave" in the best sense. It's not on anyone's roadmap, it's not strictly necessary, but the moment it exists, three or four downstream experiments become unblocked. The error explainer team has been wanting this for weeks.

I'll prototype it this week in the evenings. Goal: v0 that someone else on the team can actually run by Friday.
