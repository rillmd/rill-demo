Saturday morning, slow coffee, weekend exploration mode.

Been reading a few papers and blog posts on LLM-as-judge patterns — the idea of using an LLM to score the outputs of another LLM against a rubric. Most of the literature is about evaluating chatbot responses or summarization quality, but I keep wondering if the pattern could work for internal code review quality scoring.

Not "does the LLM replace the reviewer" — that's a dead end. More like: given a human PR review, can an LLM score *the review itself* on dimensions like specificity, actionability, whether it addresses the biggest risks? Feels like it could surface teams where reviews are pro forma rubber-stamping vs. teams where reviews are load-bearing.

The failure mode I'm worried about: the judge becomes a Goodhart target. People start writing reviews that score well instead of reviews that are actually useful. Classic metric gaming.

Maybe the right framing is "dashboard for the reviewer, never for the manager." Self-reflection tool, not a performance instrument.

Filing this away. Not an API v2 thing, but adjacent enough that it might matter later.
