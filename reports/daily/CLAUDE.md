# reports/daily/ — Daily Notes

Daily Notes generated by `/briefing`.

## Filename

`YYYY-MM-DD.md` (e.g., `2026-03-11.md`)

## Frontmatter

```yaml
---
created: 2026-03-11T08:00+09:00
type: daily-note
date: 2026-03-11
journal-count: 3
---
```

## Structure

Prose report quality. Integrates forward-looking (today's focus) + backward-looking (yesterday's activity).

Typical sections:
- **Today's Focus** — Key tasks, schedule, priorities
- **Yesterday's Review** — Completed items, progress, observations
- **Urgent Tasks** — Collected from task ticket files
- **Notes** — Other highlights

## Data Sources

Sources that `/briefing` reads:
- `inbox/journal/` recent journals
- `tasks/` urgent tickets
- `reports/daily/` recent dailies (for trend awareness)
- `knowledge/me.md` Interest Profile
- `workspace/` active workspaces

## Task Handling

Tasks are collected from ticket files (`type: task`) and presented as **summaries with background context** (ADR-063). Not a bare TODO list.

## Generation

```bash
/briefing
```

Manual creation is not recommended (loses `/briefing`'s consistent structure).

## Search Target

reports/daily/ is a Grep search target for `/focus`, `/briefing`, etc. (ADR-061). However, it is not auto-distilled into knowledge/notes/.
